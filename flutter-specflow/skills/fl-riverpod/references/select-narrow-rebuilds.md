## Riverpod: Use ref.watch(provider.select(...)) to Narrow Rebuilds

`ref.watch(provider)` subscribes to the entire state object and rebuilds the widget on any
field change. When a widget renders only one field, project it with `.select()` — the
widget then rebuilds only when that projection's `==` result changes. This matters most on
high-frequency state (camera feeds, live telemetry, large lists).

In Riverpod 3.0 all providers compare with `==` by default, so `.select` is about
**projecting a field of a larger object** — it's not needed for whole small value types
that already compare correctly.

**Incorrect:**
```dart
// ❌ full watch — rebuilds on every field change in CameraGridState
final gridState = ref.watch(cameraGridProvider);
return Text('${gridState.cameraCount} cameras'); // only uses cameraCount
```

**Correct:**
```dart
// ✅ projected — rebuilds only when cameraCount changes
final count = ref.watch(
  cameraGridProvider.select((s) => s.cameraCount),
);
return Text('$count cameras');
```

**Derived boolean** — project to a scalar so `==` short-circuits correctly:
```dart
// ✅ widget rebuilds only when the boolean flips, not on every alarm change
// .value replaces deprecated valueOrNull (Riverpod 3.0)
final hasUnacked = ref.watch(
  alarmListProvider.select(
    (async) => async.value?.any((a) => !a.acknowledged) ?? false,
  ),
);
```

**Multi-field projection** — return a record (Dart 3) or a simple value class
(`fooProvider` is generated by `@riverpod`):
```dart
final (title, count) = ref.watch(
  fooProvider.select((s) => (s.title, s.count)),
);
```

Review heuristic: any `ref.watch(provider)` in a widget that uses only one field is a
candidate for `.select()`; flag it during review. Any `.valueOrNull` is deprecated —
replace with `.value`.
